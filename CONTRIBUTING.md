# Contributing to UnifiedApp

We welcome contributions! Here's how to get started:

## Workflow

1. **Fork or Clone**: If you are a direct collaborator, clone the repo.
2. **Create a Branch**: Create a new branch for your feature or fix.
   ```bash
   git checkout -b feature/my-new-feature
   ```
3. **Make Changes**: Implement your changes.
4. **Test**: Ensure the app runs correctly (`npm run dev`).
5. **Commit**: Commit your changes with clear messages.
   ```bash
   git add .
   git commit -m "Add cool new feature"
   ```
6. **Push**: Push your branch to the remote.
   ```bash
   git push origin feature/my-new-feature
   ```
7. **Pull Request**: Open a Pull Request on GitHub.

## Coding Style
- Keep components small and focused.
- Use meaningful variable and function names.
- Formatting is handled via standard prettier/eslint settings (if configured).
